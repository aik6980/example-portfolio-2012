FXComposer ScrapBook (2011)

Last Updated on Monday, 09 July 2012 00:06 | Written by Super User | | Hits: 1497

I usually use FXComposer to learn and prototype shaders. FX Composer enables you to create advanced shading effects using .fx files, which are text descriptions of the Direct3D graphics API state and which can be loaded by any Direct3D application using the D3DX effects framework.

FXComposer is operated by an SAS script, which basically a set of annotation that we normally use to extend or add more information in HLSL script. Using the script, FXComposer will translate these into a sequence of commands which we can use to control the scene how they will be rendered.

Once you become familiar with the script, FXComposer can become a very cool tool that allow you to experiment with a number of shaders very quickly. It is very suitable for those Material shaders (applied to the object) and Image filtered Post-Processing shaders. Not only that, you can also use FXComposer to create a texture procedurally.  

Despite of it being a very useful tool. FXComposer somehow is lack of a proper documentation, which make it quite difficult to start. And since NVDia has stopped updating the tool, it also doesn't support Shader Model 5.0. I doubt we will see any better documentation.
