C++ string VS STL string?
In term of pure performance, C-string surely wins over STL because of it is just a pure array with no hidden implementation. However, using C-string is risking yourself against many problems regarding to an unprotected array, such as the buffer overflow and etc. Where STL string provided a number of safety quards, it still has the same problem as other STL container, where the underline implementation is based on using the dynamic allocation.
Uncontrollable dynamic allocation is very bad for the console development. First, lots of small dynamic allocation could cause the hidden evil, "memory fragmentation" which will slow your game down gradually through time as you call "new" operator until your game is completely crashed
STL string also have to be used with the great awareness, it might provide a number of convenience functions that help you manipulate the string around but be careful. The STL string operation usually involve the "copy" operation and it could be unnecessary costly.
For example, passing string to a function by value can be costly because if this copy operation.
The solution to the string problem is usually end where we develop the structure that is similar to the STL string with our own allocator, or using "fixed size" buffer where the memory with not be dynamically allocated but statically just like the normal array.
