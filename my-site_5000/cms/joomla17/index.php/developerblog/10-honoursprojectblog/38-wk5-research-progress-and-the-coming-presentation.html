Week 5 : Research Progress and the coming presentation

Last Updated on Monday, 02 January 2012 09:51 | | Hits: 739

This week, pretty much I still practice coding and try to organize my project in the proper way. I decided not to jump into the techniques right away. My plan is to layout some common codes which will be useful for the debugging process.

I also decide not to use my WIN32 framework and start it over from scratch. There's a lots of good programming concept and codes where the idea come from my summer placement project.

Here's a few system code I've done this week

  • RTTI
  • DirectX Renderer framework (DX10 still in progress)
  • Events system
  • Object serialization

On my research, I found the prove that the modern game can gain a huge benefit from the geometry instancing technique (AMD paper) where DX10 is considered as the main renderer. Geometry instancing is also be able to use in other situation, in order to speed up the object tessellation (still in my research progress).

Apart from this, I draw some of my focus onto the scene management technique. Mostly onto the spatial data structure, and how they gonna fit onto the LoD system.

I still try to find more research papers about the dynamic imposter. Imposter might be able to help reduce the number of rendered geometry, but they are texture memory bounded. Dynamic imposter can consume a huge amount of video memory if we don't have the right strategy on the texture memory management.

Things that might be worth looking at, in order to solve the texture memory problem

  • Mega texture
  • Texture atlas
  • Spatial data structure (Quad-tree)


In the presentation, I will try to present the techniques that is important on the modern machine, and give out the reason why DX10 is the most suitable API for the latest graphic pipeline (SM4.0)
The presentation is not meant to be very technical, but I need to make sure that i can cover all my ground work.

To do list:

  • Finish the 3rd worksheet (What? Why? How? coursework)
  • Submit the drafted presentation
