Week 6 : Directx10 cannot be defeated!

Last Updated on Monday, 02 January 2012 09:51 | | Hits: 744

This week, apart from the daily research routine, I also practically start learning DirectX10 programming. As we already knew, the DX10 architecture has been completely recreated, and most of the D3DX utillity function has been removed from the core package as well.

By then, it took me sometimes to realize those function hasn't been completely vanished, but they have been moved into DXUT library. [ oh well, thanks DirectX team :( .. ]

I'm not going to say how I integrate the DXUT with my framework, but it's working eventually. One huge restriction that DX10 users have to concern is that DXUT code is strictly using UNICODE or WCHAR, so be prepare if your app is MultiByte-based since it will not working at all.

What does the DXUT provide? Is it worth to change all your code to support UNICODE just only that package?

The answer is unclear. Anyway, here's the list of DXUT functionallity roughly,

  • SDKMESH file loader, X file loader
  • predefined DX10 intialization
  • predefined Camera
  • etc.

As far as I concerned in my app, since I'm not using the whole DXUT package so most of the DX10 wrapper are busted apart from the 3DModel Loader (Nice!). Anyway, DXUT is the open-source library, so I still benefit from learning their (DirectX people) code organization.
