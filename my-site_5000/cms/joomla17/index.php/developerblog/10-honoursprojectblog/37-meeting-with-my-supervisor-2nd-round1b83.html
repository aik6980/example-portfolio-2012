Week 4 : Meeting with my supervisor, 2nd round

Last Updated on Monday, 02 January 2012 09:51 | | Hits: 772

this is another official meeting with Matt, my project supervisor. The meeting took place in the Cafe area in the union, where we can pick a nice cup of coffee (we both love the coffee break meeting style, so we decided to keep it on like this =D).

Mainly, we has been going through some general topics about my project plan. I updated my project status, and told him about some research papers, and my DirectX10 framework. According to this, I also asked about the detail and questions that I should prepare for the coming presentation.

Next, I explained the project plan, which has been divided into the 3-4 phases. Since the project question could be able to answer from the different angles on many approaches, then I could prioritize the techniques that are most relevants. This should be able to protect myself not to overwhelm the project with too many techniques at the same time.

Here's a brief project plan,

  1. Phase 0, Reseach the available techniques and create the game framework. Mainly on DirectX10 programming and Shader 4.0 architecture
  2. Phase 1, Create the demo that represent the "geometry instancing technique", defined in "GPU Gems 2"
    Phase 1.1, Move further into the new "geometry instancing technique for Shader 4.0 Model", defined in "GPU Gems 3"
  3. Phase 2, Smart LoD System and scene management
    Static Imposter
    Animated Imposter
    Phase 2 (optional), "Can we use Geometry Shader to accelerate the imposter rendering?"
  4. Phase 3, (optional), Use multi-threaded programming technique, in order to maximize the utilization on Multi-core CPU.

The last bit of the meeting is where we discussed about the presentation. The planning part look fine so far, as long as I make it short and straight to the point.

Somehow, Matt did raise some interesting issues that I should prepare to answer,

  • "The instancing technique has been made into the last year dissertation, what does make this project interesting?"
    "Why DirectX10?"

    DirectX10 introduces the new API which is designed to be able to incooperate more efficiently on the Multi-core CPU and Shader 4.0 Model.
    Moreover, there's also the new geometry instancing technique that is specially used the Shader4.0 features (Texture Array, Vertex Shader's Lookup Textures). This is the Phase 1 target.

    In conclusion, the project will be able to show, how we could improve the instancing technique, on the modern machine (SM4.0 eg. GeForce8Series)
